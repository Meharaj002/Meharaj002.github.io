Dorpon

A premium eyewear storefront with browser-based virtual try-on and an AI concierge — and, uniquely, a machine-readable commerce layer that lets external AI agents discover and transact with the store.

The idea

Online stores are built for human eyes. Dorpon is built for both humans and the AI agents that are starting to shop on their behalf. It's a real eyewear store — catalog, cart, checkout, and a live virtual try-on that overlays frames on your face in the browser — wrapped in an AI concierge and a clean, machine-readable interface for agents.

The interesting part: one registry, three consumers

Every store capability (search, recommend, add-to-cart, checkout, start-try-on) is defined once in a single tool registry. That registry powers three surfaces simultaneously:

  • The website — an in-app AI concierge (AWS Bedrock / Claude) calls the tools.
  • An MCP server — a stateless Streamable-HTTP endpoint (spec 2025-11-25) so assistants like Claude and ChatGPT can connect and shop the store directly.
  • A UCP commerce API — a discovery manifest at /.well-known/ucp.json with JSON-Schema actions and RFC 9457 errors, aligned with the emerging Agentic Commerce Protocol.

Adding a new capability means writing one tool — it instantly appears in all three. A clean security model gates it: anonymous agents get read / discovery tools; mutating and checkout actions require a bearer token, so cross-user access is impossible by construction.

Virtual try-on

The try-on runs entirely in the browser: real-time face tracking drives a frame overlay, with predictive motion smoothing so it stays locked on even on modest hardware — no GPU server required, which keeps hosting free.

Shipped, end-to-end

Designed, built, secured, and deployed solo — frontend, backend, database, media storage, the agent protocols, and CI — on entirely free-tier infrastructure, then documented for others to extend.
